# Final Metrics Report

## Сравнение моделей

| Model             |     MAE |    RMSE | MAPE (%) |
|-------------------|--------:|--------:|---------:|
| LightGBM Baseline | 4008.35 | 5180.27 |    38.38 |
| LSTM v1           |11445.93 |13243.31 |    99.83 |

## Протокол оценки

- **Данные**: Fitbit Fitness Tracker Data (Kaggle, Mobius), 33 пользователя
- **Задача**: регрессия — предсказать `steps` на следующий день
- **Test set**: последние 20% train выборки по времени (55 записей)
- **Метрики**: MAE, RMSE, MAPE

## Выводы

### LightGBM Baseline
- MAE = 4008 шагов — ошибка ~4000 шагов при медиане ~7500 шагов
- Модель быстро обучается, хорошо использует лаговые и rolling признаки
- Распределение ошибок близко к центрированному около нуля

### LSTM v1
- MAE = 11445 — значительно хуже baseline
- MAPE = 99.83% — модель практически не обучилась
- Причина: мало данных (77 sequences для обучения), LSTM требует больше примеров
- При наличии данных за год+ LSTM может превзойти baseline

## Финальная модель в продакшн

**Выбор: LightGBM Baseline**

Обоснование:
1. MAE в 2.85 раза лучше (4008 vs 11445)
2. Быстрое обучение и инференс
3. Интерпретируемость через feature importance
4. Данных недостаточно для эффективного обучения LSTM
5. Уже задеплоен через FastAPI (`src/api/main.py`)
